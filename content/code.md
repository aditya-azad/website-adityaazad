## Personal Projects

### [Simple Static Site Generator](https://github.com/aditya-azad/simple-ssg)

This was my first stab at a transpiler and an exercise in simplicity. All the latest and greatest static site generators are either too bloated or have complex language and mechanisms to maximize flexibility. Chances are, if you are using a static site generator, you are building something simple. Why make the generator so complex? It was used to build an earlier version of this website. I have since switched to [Hugo](https://gohugo.io/) in favor of speed and reliability. I hope to build a more robust and faster version in future.

### [Destined](https://github.com/aditya-azad/destined)

A minimal task management software I built when I first learned [React](https://react.dev/) in the good old days. What makes this different is various color coding schemes, sorting mechanisms and repeatable tasks. I kept the code dependencies to a minimum to get a feel for the [JavaScript](https://www.javascript.com/) ecosystem and the "lovely" [Webpack](https://webpack.js.org/) configuration system. I don't use this anymore because of the unnecessarily complexity that I thought was necessary at the time.

### [Candle Grey](https://github.com/aditya-azad/candle-grey)

This is a colorscheme I designed for [Vim8](https://www.vim.org/vim-8.1-released.php). It was an exercise in contrast and values (color theory). I wanted to make it as simple as possible and see if I can get away with virtually no eye candy. I also wanted to force myself to write better code. Taking away the crutches of bright colors forced me to design the code in a pleasing, readable way.
