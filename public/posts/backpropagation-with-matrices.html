Neural Networks With Matrices

machine-learning

There are a lot of online resources that show how neural networks works with scalar values (probably because it's easy to understand), but there are not enough of them talking about doing it using matrices. So, in this article I will build it up, step by step, on a single dense layer.

I am assuming you understand what chain rule is, a little bit of calculus and some you have trained a neural network before, even if it was just following a tutorial. If you haven't gone through some of these, here are a few resources:

Forward pass

First, let's go through forward pass using a single neuron.

  • Each neuron in a network has a set of inputs which it receives from previous layer. We collect these in a vector and name that vector x. Each of the inputs can be represented as xi, where i is the corresponding input node.
  • For each of these inputs, we have weights associated with it which tell how important that input is for the neuron. Let's put all these weights into a vector and call it w. We use the same notation as x for weights too.
  • Let's call the neuron N.

Graphically, this is what a neuron will look like:

There three things going on inside the neuron:

  1. Weighted sum of the inputs
  2. Adding of neuron bias to the weighted sum
  3. Passing the weighted sum into the activation function

Mathematically, it looks like this:

Here ReLU (Rectified Linear Unit) is the activation function. It decides whether a neuron should fire or not. The summation part is the weighted sum that we were talking about. It is same as multiplying the input with corresponding weights and adding all the terms. Bias is used to offset the result.

Since a single neuron's weights are represented as a vector, a layer of these neurons will be represented as a matrix. Let's name that matrix W. But, what about input vector x? Since the same inputs are fed into all the neurons, it remains a vector.

When we do a dot product the input vector (left) and the weight matrix (top) we get the summations on all the neurons in the layer (middle). The diagram is represented this way to make it easier to see how the matrices are multiplied.

Here the subscript on the weights denote a neuron in the layer of n neurons. Each column in the weights matrix represent a neuron's weights. Since, each input has a corresponding weight, the number of columns in input vector is same as the number of rows in the weights matrix.

Finally, we add the bias represented as a vector B (since there are n neurons, the size of vector B is also n).

Each of the elements in the resulting vector will be passed on to ReLU activation function as per the formula above. The result of which will also be a vector of size n. This vector will represent the final output of this layer. With this, the forward pass will be complete.

There is one thing missing though. Neural networks are trained in batches. The input vector above will actually be a matrix with each row representing a input. If this was the first layer in the network, this will be equivalent to passing in a bunch of rows having i columns (features). It is the same for any hidden layer in between.

Therefore, our final output will be a matrix with each row representing the output of all the neurons given the input vectors.

As you can see, The final output looks a lot like the expression inside the ReLU equation above. Here z is the batch size.

With some numpy magic, the forward pass looks like:

import numpy as np

class Dense:

    def __init__(self, num_inputs, num_neurons):
        self.weights = np.random.randn(num_inputs, num_neurons)
        self.biases = np.random.randn(1, num_neurons)

    def forward(self, inputs):
        self.inputs = inputs
        self.output = np.dot(inputs, self.weights) + self.biases

Backpropagation

As we know, backpropagation makes use of chain rule. Essentially, to find how a variable down the line affects the final output, we have to calculate all the operations in the middle affect the final output. Here is a quick diagram:

In our case, we want to find the effect of x and w on the final ReLU output. We have two equations because we want to find the effects of both x and w on the final output. These equations are not as complex as they seem. We are just taking the individual function and chaining them. They just happen to be looking big in our case.

Furthermore, we can simplify these equations quite a bit:

  • For the third term in each equation, we know that derivative of a multiplication operation (say a x b) with respect to a variable (say a) is the leftover varaible (that is, b). So the final terms will become wi and xi repectively.
  • The second term will become 1 since bias is a constant and derivative of a term with respect to itself is 1.
  • The first term is dependent on the derivative of the activation function so we can remove that from consideration from now.
